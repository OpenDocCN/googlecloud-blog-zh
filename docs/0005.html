AWS到Google云平台服务映射

2016年9月1日更新:我们现在有一个非常全面的地图，远远超出了这篇文章的范围。你可以在这里阅读。

作为谷歌云平台团队的开发者倡导者，我经常被问到我们提供什么服务。如果与我交谈的人熟悉亚马逊网络服务(AWS)，那么开始解释谷歌云平台的最快方法是从与AWS的类似服务进行比较开始，然后涵盖差异。

下面是AWS和Google云平台中一些主要服务之间的简单映射。这并不是一个完整的映射。列出每一项服务对两个平台都不公平，因为谷歌和亚马逊在许多领域采取不同的方法，直接比较几乎是不可能的。我只列出了比较有帮助的服务。

计算:

EC2 → 计算引擎
EC2容器服务 → 容器引擎
弹性豆茎 → App引擎 **

存储:

S3 → 云存储
冰川 → 云存储近线
CloudFront→云存储(公共桶提供边缘缓存)

数据库:

RDS → 云SQL
T5】DynamoDB→云Datastore 和云Bigtable

大数据:

红移→big query
SQS/Kinesis→云Pub/Sub
EMR→云数据流

监控:

CloudWatch → 云监控和云日志

网络:

Route53 → 云DNS 和谷歌域名
直连 → 云互联

其他:

CloudFormation → 云部署管理器
T5】SES→send grid(合作伙伴)
WorkMail→Gmail(另见Google for Work)
Work Docs→Google Docs(另见 Google for Work )

** AWS Elastic Beanstalk和Google App Engine经常被描述为类似的产品，但它们的方法有显著的差异。两者都提供自动伸缩、负载平衡、监控等功能。但是与App Engine不同，Elastic Beanstalk需要原始虚拟机所需的典型系统管理(操作系统更新等)。).Google App Engine是一个PaaS，这意味着它是完全托管的，因此所有这些管理任务都由Google处理。基本的应用程序引擎设置包括内置服务，如任务队列、Memcache、用户API等。

如果您需要未受管理的虚拟机，Google还有自动扩展、负载平衡和监控未受管理的虚拟机作为 Google计算引擎的功能。现在还有一种替代的托管模式，作为谷歌应用引擎的一部分，称为托管虚拟机。

我的建议是，在进入任何一个平台之前，都要做足功课，彻底理解这些模型。各有各的优势。
